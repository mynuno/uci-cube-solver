# UCI Cube Solver

## 1. 프로젝트 목표

UC Irvine Anteaters Puzzle Cube Keychain을 사진 기반으로 분석하고,
사용자가 큐브 표기법을 몰라도 3D 애니메이션을 따라 실제 큐브를 맞출 수 있게 한다.

최종 흐름:

1. 사용자가 큐브 6면을 촬영
2. 각 사진을 3×3 조각으로 분할
3. 사용자가 각 조각의 목표 면을 지정
4. 프로그램이 목표 그림 배열을 구성
5. 현재 큐브 상태를 검증
6. 풀이 수열 계산
7. 사진 텍스처가 적용된 3D 큐브로 회전 동작 안내

---

## 2. 확정된 목표 면 구성

- F: 흰색 바탕 / 남색 로고
- U: 노란색 바탕 / 남색 로고
- R: 남색 바탕 / 노란색 로고
- B: 흰색 바탕 / 남색 글씨
- D: 노란색 바탕 / 남색 글씨
- L: 남색 바탕 / 흰색 글씨

F, U, R, B, D, L은 프로그램 내부 좌표로 사용한다.

사용자에게는 가능하면 표준 큐브 기호보다 3D 애니메이션과 자연어 안내를 제공한다.

---

## 3. 현재 확인된 제약

- 6면 전체의 공장 출고 상태 사진은 없다.
- 제품 사진으로 일부 면의 방향만 확인할 수 있다.
- 사용자가 조각을 보고 어느 목표 면에 속하는지는 구분할 수 있다.
- 사용자가 각 조각의 회전 각도를 직접 입력하는 것은 어렵다.
- 조각의 목표 방향과 위치는 프로그램이 추론하거나 사용자가 후보를 확인한다.
- 일부 조각은 시각적으로 비어 있지만 로고 또는 글씨 구조로 구분 가능하다.

---

## 4. 설계 원칙

- 큐브 계산 로직과 화면 표시 로직을 분리한다.
- 실제 물리 구조에 맞게 Cubie 중심으로 설계한다.
- Sticker는 Cubie에 소속된다.
- 이미지 원본과 잘린 조각 이미지는 별도 저장소에서 관리한다.
- Sticker는 이미지 자체 대신 imageId를 참조한다.
- 현재 상태와 목표 상태를 분리한다.
- 사람이 판단하기 어려운 회전 정보는 입력값으로 요구하지 않는다.
- 모바일 촬영과 Google Drive 연동을 고려해 웹앱으로 개발한다.

## 5. 데이터 구조 방향

### Cube
- 8 Corner Cubies
- 12 Edge Cubies
- 6 Center Cubies

총 26개의 Cubie로 구성한다.

### Cubie
- 고유 ID
- 종류: corner, edge, center
- 현재 위치
- 현재 방향
- 소속 Sticker 목록

### Sticker
- 고유 ID
- 현재 바라보는 면
- 목표 면
- imageId
- 목표 위치
- 목표 방향

### Image Asset
- 고유 ID
- 이미지 출처
- 원본 또는 잘린 이미지 URL
- 촬영된 면
- 행과 열
- 원근 보정 및 크롭 정보

## 8. 촬영 기준

기준 자세:

- U는 위쪽
- F는 정면
- R은 오른쪽

촬영 규칙:

- F, R, B, L: U와 맞닿는 변을 사진 위쪽에 둔다.
- U: F와 맞닿는 변을 사진 아래쪽에 둔다.
- D: F와 맞닿는 변을 사진 위쪽에 둔다.

그림이 똑바로 보이는 방향이 아니라 물리적 면 관계를 기준으로 촬영한다.
