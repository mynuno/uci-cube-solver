# Current Task

## 현재 작업

모바일 화면에서 이전·다음 단계 이동을 항상 사용할 수 있도록 하단 고정 내비게이션을 구현한다.

## 시작 상태

* 프로젝트 상태와 이벤트 핸들러는 `src/store/useCubeProject.ts`에서 관리
* 순수 CubeState 계산과 복제 로직은 `src/cube/projectState.ts`로 분리
* 사진 등록, 조각 분류, 완성 검증, 풀이 안내 화면은 `src/components/workflow` 아래의 단계별 컴포넌트로 분리
* `App.tsx`는 공통 헤더, 단계 내비게이션, 공통 알림과 단계 컴포넌트 조합을 담당
* 데스크톱 화면 상단에 4단계 워크플로 내비게이션 존재
* 화면 하단에 이전 단계·현재 단계·다음 단계 내비게이션 존재
* JSON `schemaVersion: 1` 호환성 유지
* cubejs 풀이 생성과 완료 화면 정상 작동
* Phase 1 구조 안정화 작업 완료
* lint, build, test 통과

## 이번 작업 목표

* 모바일 화면에서 이전 단계와 다음 단계 버튼을 화면 하단에 고정
* 긴 화면을 스크롤한 상태에서도 단계 이동 버튼을 즉시 사용할 수 있도록 개선
* 현재 단계와 전체 단계 수를 하단 내비게이션에서 확인 가능하도록 유지
* 첫 번째 단계에서는 이전 버튼을 사용할 수 없도록 유지
* 마지막 단계에서는 다음 버튼을 사용할 수 없도록 유지
* 모바일 안전 영역과 브라우저 하단 UI를 고려한 여백 적용
* 고정 내비게이션이 본문이나 버튼을 가리지 않도록 하단 여백 확보
* 데스크톱의 기존 상단 단계 내비게이션과 화면 배치를 유지
* 기존 단계 이동 함수와 상태 전이를 그대로 사용

## 모바일 동작 기준

* 모바일 기준 너비는 기존 미디어 쿼리와 일치시킨다.
* 하단 내비게이션은 모바일 화면에서만 고정한다.
* 하단 내비게이션의 z-index는 일반 콘텐츠보다 높고 모달보다 낮아야 한다.
* `env(safe-area-inset-bottom)`을 고려한다.
* 내비게이션 높이만큼 본문 하단에 padding을 추가한다.
* 이전·다음 버튼의 비활성화 상태가 시각적·기능적으로 유지돼야 한다.
* 버튼의 접근 가능한 이름과 키보드 포커스 동작을 유지한다.
* 단계 이동 후 기존과 같이 화면 상단으로 스크롤한다.
